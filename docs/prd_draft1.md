# 영어 작문 AI 스캐폴딩 튜터 - 제품 요구사항 정의서 (PRD) 초안 1

본 문서는 영어 작문 교육 최신 연구 동향(지능형 튜터링 시스템, Fading 스캐폴딩, 과의존 방지, 피드백 리터러시)을 통합하여 설계된 **영어 작문 AI 스캐폴딩 튜터(English Writing Scaffold Tutor)**의 제품 요구사항 정의서 초안입니다.

---

## 1. 프로젝트 개요

### 1.1 프로젝트명
* **영어 작문 AI 스캐폴딩 튜터 (English Writing Scaffold Tutor)**

### 1.2 프로젝트 목적
본 시스템은 완성형 정답 번역문을 학습자에게 단순히 주입하는 기존의 자동 작문 첨삭기(AWE) 모델에서 벗어나, 학습자가 스스로 영어 문장을 생각하고 작성해 나갈 수 있도록 **진단 ➡️ 힌트 ➡️ 재시도 ➡️ 반성 ➡️ 점진적 지원 축소(Fading)**를 유도하는 지능형 튜터링 시스템(ITS)을 구축하는 것을 목적으로 합니다.

### 1.3 핵심 철학
* **정답 직접 제시 절대 금지**: AI는 완성된 정답 문장을 학생에게 절대 노출하지 않습니다.
* **스캐폴딩 기반의 조력(Fading)**: 학생이 막힐 때만 점진적으로 힌트를 강화하되, 최종 문장은 반드시 학생 스스로 작성하게 합니다.
* **피드백 리터러시와 자기반성**: 학생이 힌트를 통해 스스로 수정한 과정을 인지하게 돕습니다.
* **교사의 보완적 모니터링**: 튜터링의 사각지대(과의존 학생, 반복 오류 학생)를 교사가 실시간으로 포착하도록 돕습니다.

### 1.4 핵심 아키텍처 (ITS 구조)
본 시스템은 ITS 연구의 4대 핵심 요소를 충족합니다.

| ITS 요소 | 시스템 내 구현 정의 |
| :--- | :--- |
| **Domain Model (도메인 모델)** | 교육부 초등 영어 성취 기준, 학년별 필수 어휘/문법 데이터, AI 프롬프트 규칙 및 정답 차단 정책 |
| **Student Model (학생 모델)** | 학생 정보(학년, 반, 이름, 수준) 및 동적 학습 데이터(힌트 요청 단계, 누적 오류 카테고리, 수정 횟수, 과의존 여부) |
| **Tutor Model (튜터 모델)** | 스캐폴딩 단계(1~5단계) 선택 규칙, 힌트 생성 프롬프트, 피드백 제어 및 과의존 감지 알고리즘 |
| **User Interface (UI 모델)** | 작문창, 힌트 카드, 실시간 수정 비교 뷰, 자기반성 입력 영역, 교사 대시보드 |

## 2. 핵심 기능 요구사항

### 2.1 학생 정보 등록 및 작문 시작
* 학년, 반, 이름 입력
* 난이도 레벨 설정 (초등 3~4학년 / 초등 5~6학년 / 중학생)
* 표현하고자 하는 우리말 문장 입력

### 2.2 실시간 작문 대화 및 단계별 스캐폴딩 (Scaffolding Loop)
학생이 문장을 입력하거나 "힌트 더 받기"를 요청할 때, 튜터 정책 엔진은 순차적으로 아래의 힌트를 제공합니다. 단, 학생이 다음 문장을 입력하거나 직접 고쳐보기 전에는 다음 힌트 단계로 넘어가는 것이 잠깁니다.

1. **1단계 (의미 확인 - Basic)**
   * 영어 문장을 일절 배제한 채 문장 속 주어("누가"), 동사("무엇을 하는지"), 수식어("어디서/언제") 등 의미 구조를 한글로 파악하게 돕습니다.
2. **2단계 (오류 위치 표시)**
   * 학생이 첫 문장을 시도한 후, 틀린 부분이 있는 위치를 짚어줍니다 (예: "세 번째 단어 부근의 형태를 다시 확인해보세요").
3. **3단계 (문법 개념 힌트)**
   * 문법적으로 보완해야 할 개념을 설명합니다 (예: "어제 일어난 일이니, 동사 뒤에 -ed를 붙여서 과거형으로 써봐요").
4. **4단계 (단어 카드 제시)**
   * 문장에 필요한 핵심 어휘(예: play, yesterday 등)를 단어 카드 형태로 동적으로 제공합니다.
5. **5단계 (빈칸 문장틀 - Scaffolding Frame)**
   * 학생이 최종적으로 문장을 완성할 수 있도록 뼈대를 제공합니다. (예: `I + play__ + soccer + yesterday.`)

### 2.3 AI 튜터 정책 엔진 & 출력 검증기 (Output Validator)
* **정답 차단 필터**: AI의 응답 텍스트에 영어 완성 문장이 포함되어 나가는 것을 방지합니다. 정규식과 키워드를 기반으로 감시하여, 노출 시 대체 멘트("직접 완성할 수 있도록 힌트를 줄게요")로 자동 필터링합니다.
* **수준 제약 필터**: 학년별 수준을 초과하는 문법 용어(예: 초등 3~4학년에게 '동명사', '수동태' 등)나 어휘를 힌트 및 피드백에서 제외하고 대체 설명으로 변경합니다.

### 2.4 과의존 탐지 (Overdependence Detection)
* **규칙**: 한 세션에서 **4단계(단어 카드) 또는 5단계(빈칸 문장틀) 힌트를 연속으로 2회 이상 사용**한 경우, 학생의 학습 태도를 과의존 상태로 판단합니다.
* **액션**: 
  - 세션 데이터에 `is_overdependent = TRUE` 플래그를 저장합니다.
  - 교사 대시보드에서 해당 학생의 레코드 옆에 경고 플래그(예: ⚠️ 과의존 주의)를 표시합니다.

### 2.5 피드백 리터러시 (자기점검 및 반성 루프)
* 작문이 완료되었을 때 바로 종료되지 않고, 학생이 **자신의 이전 문장과 최종 완성 문장을 대조하는 화면**을 거쳐야 합니다.
* 학생은 **"AI 힌트를 통해 내가 스스로 고친 부분과 새롭게 배운 점"**을 한글로 직접 입력하는 자기반성 질문(`reflection_text`)에 답변해야 세션이 최종 종료됩니다.

### 2.6 교사 대시보드 (`/teacher`)
* **목표**: 교사가 학급 학생들의 작문 진행률과 튜터의 보완 사각지대를 관찰할 수 있도록 지원합니다.
* **포함 정보**:
  * 학생별 기본 인적사항 및 난이도 수준
  * 학생별 누적 작문 세션 수, 세션당 평균 수정 횟수
  * **과의존 탐지 학생 리스트** (⚠️ 4~5단계 힌트 연속 2회 이상 사용자 시각적 강조)
  * 학생별 자주 틀리는 오류 카테고리 (어휘, 시제, 명사 복수형 등)
  * 학생들이 작성한 자기반성문(`reflection_text`) 모아보기

## 3. 데이터베이스 스키마 수정안

### 3.1 `students` 테이블
*(변동 없음)*

### 3.2 `writing_sessions` 테이블 (수정 및 추가)
* `reflection_text TEXT`: 피드백 리터러시를 위한 학생의 자기반성 글
* `is_overdependent BOOLEAN DEFAULT FALSE`: 최종 힌트 과의존 여부 플래그
```sql
ALTER TABLE writing_sessions ADD COLUMN reflection_text TEXT;
ALTER TABLE writing_sessions ADD COLUMN is_overdependent BOOLEAN DEFAULT FALSE;
```

### 3.3 `conversation_logs` 테이블 (수정 및 추가)
* `detected_errors TEXT[]`: 해당 대화 턴에서 AI가 학생 문장에서 탐지한 문법 오류 카테고리 배열
```sql
ALTER TABLE conversation_logs ADD COLUMN detected_errors TEXT[];
```
